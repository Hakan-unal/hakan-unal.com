Using the File System Module in Node.js (With Samples)

File system related operations of Node.js platform are done through fs module. Node.js works asynchronously the fs module works asynchronously accordingly but almost all functions have synchronous versions.

Using the fs module, we can implement following operations:

  • Opening File
  • Writing File
  • Reading File
  • Removing File

Opening File

fs.open() method is used to open files asynchronously and fs.openSync() method is used to open files synchronously on your computer. fs.open() has at least 3 parameters:

1. Parameter : File path
2. Parameter : Flag
3. Parameter : calback function

Let's continue with code sample below but first, let's create the test.txt in the directory where the JavaScript file is path

// Included the fs module to file
const fs = require("fs");

fs.open("test.txt" , "r+" , (err,fd) => {

if(err){
console.log(err);
}

else{
console.log("File successfully opened");
}

});

There is an important point I would like to mention here. You can see what the flags are doing on the table below:

  • r Open for reading. An exception occurs if the file does not exist.
  • r+ Open for reading and writing. An exception occurs if the file does not exist.
  • rs Open for reading in synchronous mode, which instructs the operating system to bypass the system cache. This is mostly used for opening files on NFS mounts; it does not make open() a synchronous method.
  • rs+ Open for reading and writing in synchronous mode.
  • w Open for writing. If the file does not exist, it is created. If the file already exists, it is truncated.
  • wx Similar to the w flag, but the file is opened in exclusive mode. Exclusive mode ensures that the files are newly created.
  • w+ Open for reading and writing. If the file does not exist, it is created. If the file already exists, it is truncated.
  • wx+ Similar to the w+ flag, but the file is opened in exclusive mode.
  • a Open for appending. If the file does not exist, it is created.
  • ax Similar to the a flag, but the file is opened in exclusive mode.
  • a+ Open for reading and appending. If the file does not exist, it is created.
  • ax+ Similar to the a+ flag,but the file is opened in exclusive mode.

Writing File

fs.writeFile() method is used to write files asynchronously and fs.readFileSync() method is used to write files synchronously on your computer. fs.writeFile() function has at least 3 parameters:

1. Parameter : File path
2. Parameter : Data to be written into the file
3. Parameter : callback function
Optional Parameter : Encoding type

Let's start by doing an sample together

// Included the fs module to file
const fs = require("fs");

// Data to be written into the file
const data = "My First Sentence";

fs.writeFile("myFirstText.txt" , data , (err) => {
if(err){
console.log(err);
}

else{
console.log("Succesful operation!");
}

});

It should definitely not be forgotten. If the file path is to be written, it should be written starting from the root directory. If you use the path module for this, it is easier.

Reading File

fs.readFile() method is used to read files asynchronously and fs.readFileSync() method is used to read files synchronously on your computer. fs.readFile() function has at least 2 parameters:

1. Parameter : File path
2. Parameter : callback function
Optional Parameter : Encoding type

Let's start fast together and create the file named fs_first_try.txt file and get the content as follows:

"Hello World, Welcome to Node.js"

However, let's create fs_first_try.js file and get the content as follows:

// Included the fs module to file
const fs = require("fs");

fs.readFile("fs_first_try.txt" , (err,data) => {

if(err){
console.log(err);
}

else{
console.log(data.toString());
}

});

Congratulations! You performed the first successful operation with the fs module. When the above block of code runs, you will see the "Hello World, Welcome to Node.js" output in the consol

Removing File

fs.unlink() method is used to remove file asynchronously and fs.unlinkSync() method is used to remove file synchronously on your computer. fs.unlink() function has at least 2 parameter:

1. Parameter : File path
2. Parameter : callback function

These functions take the file path as a parameter. However, the asynchronous version also takes a callback function as a parameter. It should definitely not be forgotten. If the file path is to be written, it should be written starting from the root directory. If you use the path module for this, it is easier.Let's continue with code sample below:

// Included the fs and path module to file
const fs = require("fs");
const path = require("path");

// We got the path of the file starting in the root directory
filePath = __dirname + path.sep + "myFile.txt";

fs.unlink(filePath, (err) => {

if(err){
console.log(err);
}

else{
console.log("File deleting succesful");
}

});
